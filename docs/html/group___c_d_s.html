enum cds_result

Enum for return values describing the success/failure of a function.

Todo:
Had an idea the other day that this enum is a bunch of bitfields and ORing them could reveal multiple errors. But this may be difficult to implement, and would conflict with the idea of early returns?
Enumerator:
CDS_UNIMPLEMENTED 

A feature that's not yet supported.

CDS_DUPLICATE_VALUE 

A duplicate insertion into a set like structure.

CDS_BAD_ALLOC 

A failed memory allocation.

CDS_OVERFLOW 

An overflow of insertions into a container.

CDS_UNDERFLOW 

An underflow of removals from a container.

CDS_INDEX_OUT_OF_RANGE 

An index outside the bounds of a container.

CDS_NULL_ARGUMENT 

An invalid NULL pointer argument.

CDS_INVALID_OPERATION 

A generic return for improper operations.

CDS_INVALID_ARGUMENT 

A generic return for imporper arguments passed to a function.

CDS_NOT_FOUND 

Indicates the search value was not found in the container (not a runtime error)

CDS_OK 

Function completed successfully.
